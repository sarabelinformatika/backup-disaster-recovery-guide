# Contributing

Thank you for your interest in improving this project.

Contributions are welcome.

## Ways to Contribute

Examples include:

- Technical corrections
- Documentation improvements
- Architecture suggestions
- Enterprise best practices
- Backup and disaster recovery recommendations
- Diagram improvements
- Grammar corrections

## Before Opening a Pull Request

Please ensure that:

- Information is technically accurate.
- Recommendations follow enterprise best practices.
- Markdown formatting is preserved.
- Existing writing style is maintained.

## Writing Style

This guide focuses on:

- vendor-neutral concepts;
- enterprise architecture;
- operational best practices;
- long-term maintainability.

Avoid product-specific configuration unless it supports a broader architectural principle.

## Reporting Issues

Bug reports, documentation improvements and feature suggestions can be submitted through GitHub Issues.

Thank you for helping improve this guide.
